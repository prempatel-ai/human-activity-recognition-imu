## The Problem

<br/>

```
              WHAT THE PHONE KNOWS
              в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ

   Every 1/50th of a second, the phone writes down 9 numbers.

   time=0.00s:  [ 0.12,  9.79,  0.34,  0.02, -0.01,  0.00,  0.14,  9.81,  0.33 ]
   time=0.02s:  [ 0.15,  9.81,  0.31,  0.01, -0.02,  0.01,  0.17,  9.80,  0.30 ]
   time=0.04s:  [ 0.89,  9.45,  0.67,  0.12, -0.08,  0.03,  0.91,  9.43,  0.65 ]
   ...

   After 2.56 seconds (128 readings), one question:

              What was this person doing?
```

<br/>

The phone has no camera. No GPS. No microphone. Just numbers вҖ” forces and rotations recorded by an accelerometer and a gyroscope.

From those numbers alone, this model answers: **walking, walking upstairs, walking downstairs, sitting, standing, or laying down** вҖ” with 93.15% accuracy on people it has never seen before.

Four of the six classes are straightforward. Walking creates a rhythmic spike pattern вҖ” every footstep leaves a signature in the accelerometer data. Laying down shifts gravity from one axis to another. These are easy.

SITTING and STANDING are not. Both produce nearly flat, near-zero signals. Both have gravity acting on the same axis. Their cosine similarity is 0.97 out of 1.0 вҖ” they are 97% identical to standard mathematical analysis. The entire model architecture вҖ” specifically the attention mechanism вҖ” exists primarily to resolve this one problem.

## Technical Pipeline

<br/>

### 1 вҖ” Signal Analysis (FFT)

<br/>

Before any modelling, we analyse what information the signals actually carry.

The **Fast Fourier Transform** converts a time-domain signal (value vs time) to a frequency-domain spectrum (strength vs frequency):

```
FFT formula:  X[k] = sum_{n=0}^{N-1}  x[n] * exp(-j * 2*pi * k * n / N)

In plain English: for each frequency k, multiply the signal by a reference
wave at that frequency and sum all the products.
Large result = that frequency is present. Near zero = it is not.
```

**What FFT reveals:**

```
Walking signal FFT:             Sitting signal FFT:

Strength                        Strength
   |                               |
   |    *                          | * * * * * * * * * * * *
   |    |  *                       |_____________________________
   |    |  |                       0                         25 Hz
   |____|__|____
        2  4  Hz
        вҶ‘
   Strong peak at ~2 Hz        No dominant frequency anywhere
   = footstep rhythm            = no rhythm, just noise
```

**Frequency resolution** = 50 Hz / 128 samples = 0.39 Hz per bin.

**The critical finding:** SITTING and STANDING have identical FFT spectra вҖ” both flat. The model must find subtle time-domain differences. This is why we need the LSTM and attention, not just FFT-based features.

<br/>

### 2 вҖ” Preprocessing

<br/>

**StandardScaler normalisation:**

Raw channel ranges differ by 5-10x. Without normalisation, the model learns that channels with larger numbers matter more вҖ” which is wrong.

```
Before:  Body Acc X range  вүҲ 2.5   (small numbers)
         Body Gyro Y range вүҲ 12.0  (large numbers)

The model would think Gyro matters 5x more вҖ” just because numbers are bigger.

After StandardScaler:
   x_scaled = (x - mean_channel) / std_channel

All channels:  mean = 0.0,  std = 1.0
Model treats all channels as equally important by default.
```

**Critical rule:** `scaler.fit_transform(X_train)` then `scaler.transform(X_test)`.
Never fit on test data вҖ” that is data leakage.

**One-hot encoding:**
```
Label 0  вҶ’  [1, 0, 0, 0, 0, 0]   WALKING
Label 3  вҶ’  [0, 0, 0, 1, 0, 0]   SITTING
Label 5  вҶ’  [0, 0, 0, 0, 0, 1]   LAYING
```
Required because the model outputs 6 probabilities вҖ” targets must match format.

<br/>

### 3 вҖ” CNN: Local Pattern Detection

<br/>

A 1D convolutional layer slides a small filter across the time axis:

```
Convolution at position t:

output[t] = filter[0] * signal[t]
           + filter[1] * signal[t+1]
           + filter[2] * signal[t+2]
           + bias

filter size=3 at 50Hz = 60ms window per computation
```

With 64 filters in Block 1, the layer runs this 64 times with different learned weights. After training:
- Some filters detect footstep spikes
- Some detect zero crossings
- Some detect sustained flatness

MaxPooling(2) halves the sequence (128 вҶ’ 64 вҶ’ 32), expanding the effective receptive field:

```
Block 1 effective window:  3 steps Г— 20ms  =  60ms
Block 2 effective window:  9 steps Г— 20ms  = 180ms
```

<br/>

### 4 вҖ” LSTM: Temporal Pattern Recognition

<br/>

The LSTM reads 32 compressed feature vectors in sequence. At each position, three gates control what to remember:

```
Forget gate:  f_t = sigmoid(W_f В· [h_{t-1}, x_t] + b_f)
              What to erase from long-term memory

Input gate:   i_t = sigmoid(W_i В· [h_{t-1}, x_t] + b_i)
              What new information to write

Cell update:  c_t = f_t * c_{t-1} + i_t * tanh(W_c В· [h_{t-1}, x_t] + b_c)
              Keep old + write new

Output gate:  h_t = sigmoid(W_o В· [h_{t-1}, x_t] + b_o) * tanh(c_t)
              What to expose as current output
```

The cell state `c_t` flows through with only additive updates вҖ” no repeated weight multiplication. This prevents the vanishing gradient problem that breaks vanilla RNNs, allowing the LSTM to learn dependencies across all 32 positions.

For walking: LSTM accumulates evidence of regular footstep intervals.
For static: LSTM accumulates evidence of sustained near-zero activity.

<br/>

### 5 вҖ” Training

<br/>

**Loss function вҖ” categorical cross-entropy:**

```
L = -log(probability assigned to true class)

Model gives 90% to correct class:  L = -log(0.90) = 0.105   (small penalty)
Model gives 10% to correct class:  L = -log(0.10) = 2.303   (large penalty)
```

Heavy penalty for confident wrong predictions drives the model to be both accurate and calibrated.

**Optimiser вҖ” Adam:**

```
m_t = 0.9 * m_{t-1} + 0.1 * g_t        (momentum вҖ” running average of gradients)
v_t = 0.999 * v_{t-1} + 0.001 * g_t^2  (adaptive scale вҖ” running average of squared gradients)

w = w - (lr / sqrt(v_t) + eps) * m_t

Effect: large consistent gradients get smaller effective steps
        small inconsistent gradients get larger effective steps
        faster, more stable convergence than vanilla SGD
```

## The Attention Mechanism

<br/>

This deserves its own section because it is the architectural choice that separates this model from a standard CNN-LSTM baseline.

**The problem it solves:**

A standard sequence classifier takes only the final LSTM hidden state `h_32`.
This implicitly assumes the last timestep summarises everything relevant.
For SITTING vs STANDING, the discriminating signal вҖ” tiny postural sway вҖ” may be
concentrated in 3-4 specific timesteps buried in a 32-timestep sequence.
Using only `h_32` dilutes this signal across all timesteps equally.

**The solution вҖ” three steps:**

```
Step 1: Score each of 32 hidden states
        e_t = tanh(w_a * h_t + b_a)
        One number per timestep. tanh keeps scores in [-1, +1].

Step 2: Normalise scores to attention weights summing to 1.0
        alpha_t = exp(e_t) / sum(exp(e_s) for all s)
        These are the attention weights вҖ” a probability distribution over timesteps.

Step 3: Weighted sum to produce context vector
        c = sum(alpha_t * h_t)
        Timesteps with high alpha contribute more to the final representation.
```

**The cost:**

```
Dense(1) on 128 inputs = 128 weights + 1 bias = 129 parameters

129 parameters = 0.05% of total model size
Yet improves STANDING recall by approximately 4 percentage points.
```

This asymmetry вҖ” negligible parameter cost, meaningful accuracy improvement вҖ” is the hallmark of a well-targeted architectural inductive bias. The attention mechanism tells the model where to look. Without it, the model has to spread its attention equally across all timesteps, diluting the subtle evidence.
